This website was created in Java using the Spring Boot framework, along with additions such as Thymeleaf (front-end to back-end integration) and Bootstrap (HTML page styling and responsiveness).

  • Main project: Our main project is an image hosting and sharing site similar to Instagram. After registering for an account, users can post images to be viewed on the image feed, as well as comments that can be posted on others' posts. Posts and comments can also be deleted by users.
  • Mini-labs: Each team member has worked on their own set of mini-labs, with each demonstrating a specific competency in Java/CS principles.
  • REST API: Developers have the ability to view data associated with posts, comments, and users via custom-created endpoints in either a tabular format or a JSON format. The Misc tab provides a number of pages where users can visualize our data in a tabular format; users can also interact with the REST API of Snake Eyes, our crossover team.
  • Misc: Other components of our project include our doodle jump game and this “how it’s made” page.

Our project was worked on collaboratively and asynchronously via the version control platform GitHub and the code processor IntelliJ. On GitHub, we maintain a readme file detailing various technicals and completions, as well as a scrum board, which allowed us to plan and coordinate tasks.

Main Section Example #1: User Controller

This file consists of all the endpoints related to the main project, ranging from creating an account to deleting a post. We make use of a variety of different mappings (POST, GET), some with PathVariables and others with RequestParams. This file has a number of dependencies (e.g. UserService, SecurityService) that handle database interaction and user session login.

Main Section Example #2: Post Model

This file highlights the Post POJO, which is specifically set up to be used in our database, that is a centerpiece for our main project. It consists of fields including the ID of comments linked to it and the time it was posted. We make use of the CLOB data type to store long strings in our database (as opposed to strings, which are limited to 255 characters).

REST API Example: REST API Controller

This file consists of all the endpoints related to the REST API—both for our REST API (that exposes information to be viewed by developers regarding posts, comments, and users) and that of our crossover group’s API. For our REST API, the endpoints prefixed with an underscore return JSON information, while those not return information optimized for our interactive, tabular viewer.

Mini-lab Example: GCD

This class contains a recursive algorithm to calculate the greatest common divisor between two integers.
